## Overview

Created a **fully containerized automated testing platform** deployed on Kubernetes — the most scalable and production-grade test infrastructure in this portfolio. Tests run on Kubernetes pods that spin up on demand and terminate automatically after execution, making the system both efficient and cost-effective.

## Why Kubernetes?

Docker Compose is great for local setups, but it doesn't scale automatically. Kubernetes allows the test infrastructure to spin up exactly as many pods as needed based on load — making this solution enterprise-ready and suitable for large test suites.

## What I Built

### Containerized Test Framework
- Packaged the entire automation framework into a **Docker image** with all dependencies included
- Multi-stage `Dockerfile` to keep the final image size minimal and efficient
- Images pushed to a **container registry** (Docker Hub / AWS ECR) after every build

### Kubernetes Deployment
- Wrote **Kubernetes manifests** (`Deployment`, `Job`, `ConfigMap`) to define the test infrastructure
- Used **Kubernetes Jobs** for one-time test execution — pods run tests, then terminate automatically
- Configured **resource limits** on pods to prevent runaway processes from affecting the cluster

### Jenkins + Kubernetes CI/CD Pipeline
- Jenkins pipeline automatically **builds a fresh Docker image** on every code change
- Pipeline deploys the updated image to the Kubernetes cluster and triggers test execution
- After execution, pod logs are collected and **Allure report is generated** — fully hands-free

## Results

- **100% reproducible** test environments — identical execution every time, no dependency drift
- Tests scale horizontally by simply increasing pod replica count
- CI/CD pipeline runs **end-to-end with zero manual steps**
